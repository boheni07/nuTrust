## 1. Executive Summary

| Perspective | Description |
|-------------|-------------|
| **Problem** | 고객 요청이 이메일, 전화, 메신저 등에 분산되어 추적이 불가능하고, 처리 과정이 불투명하여 고객 신뢰가 저하된다. 내부 지원팀은 요청 배정과 우선순위 판단에 비효율이 발생하며, 처리 완료 후 고객 승인과 만족도 수집 프로세스가 부재하여 서비스 품질 개선의 피드백 루프가 끊어져 있다. |
| **Solution** | 통합 티켓 관리 플랫폼을 통해 고객 요청의 등록부터 배정, 처리, 승인, 만족도 평가까지 전체 라이프사이클을 단일 시스템에서 관리한다. 실시간 상태 추적과 양방향 소통 채널을 제공하여 처리 과정의 투명성을 확보하고, 구조화된 승인/CSAT 워크플로우를 통해 지속적 서비스 품질 개선을 가능하게 한다. |
| **Function UX Effect** | 고객은 셀프서비스 포털에서 티켓을 등록하고 실시간으로 처리 상태를 확인하며, 완료 시 직접 승인/반려하고 만족도를 평가한다. 지원 담당자는 대시보드에서 배정된 티켓을 처리하고 내부 메모와 고객 소통을 한 화면에서 수행한다. 운영 관리자는 SLA 모니터링, 팀 성과 분석, 워크플로우 자동화를 통해 서비스 운영을 최적화한다. |
| **Core Value** | **신뢰 구축 (Trust Building)** -- 요청 처리의 전 과정을 투명하게 공개하고, 고객이 직접 결과를 승인하는 구조를 통해 고객과 서비스 조직 간의 신뢰를 체계적으로 구축한다. 이는 단순한 티켓 관리를 넘어 "고객 승인 기반 서비스 완결성"이라는 차별화된 가치를 제공한다. |

## 2. Problem Statement

### 2.1 고객 고통 (Customer Pain Points)

#### Pain Point 1: 요청 추적 불가능 (Request Tracking Blindspot)

고객이 이메일, 전화, 메신저 등 다양한 채널을 통해 요청을 보내지만, 해당 요청이 접수되었는지, 누가 처리하고 있는지, 언제 완료될지 알 수 없다. 고객은 "내 요청이 어떻게 되고 있나요?"를 반복 문의하게 되고, 이는 양측 모두에게 비효율과 불만을 야기한다.

- **빈도**: 모든 요청의 60-70%에서 고객이 진행 상황을 별도 문의
- **영향**: 고객 이탈율 증가, 반복 문의로 인한 지원팀 업무량 30% 증가
- **감정**: 불안, 불신, 좌절

#### Pain Point 2: 처리 과정의 불투명성 (Opaque Resolution Process)

요청이 접수된 후에도 내부적으로 어떤 단계에 있는지, 왜 지연되고 있는지에 대한 정보가 고객에게 전달되지 않는다. "블랙박스" 상태에서 결과만 통보받는 경험은 서비스에 대한 신뢰를 근본적으로 훼손한다.

- **빈도**: 처리 기간이 2일 이상인 요청의 80% 이상
- **영향**: NPS(Net Promoter Score) 20-30점 하락 요인
- **감정**: 소외감, 통제력 상실

#### Pain Point 3: 완료 기준의 불일치 (Resolution Criteria Mismatch)

지원팀이 "완료"라고 판단한 시점과 고객이 "해결되었다"고 느끼는 시점이 다르다. 고객 승인 프로세스가 없으면 일방적 완료 처리가 반복되고, 이는 재문의(reopen)와 만족도 하락으로 이어진다.

- **빈도**: 처리 완료 티켓의 25-35%가 재문의
- **영향**: 재작업(rework) 비용 증가, 고객 만족도 저하
- **감정**: 무시당함, 의견이 반영되지 않음

#### Pain Point 4: 서비스 품질 개선 데이터 부재 (No Feedback Loop)

만족도 조사가 체계적으로 이루어지지 않아, 어떤 유형의 요청에서 고객이 불만족하는지, 어떤 담당자의 서비스가 우수한지에 대한 데이터가 축적되지 않는다. 감(感)에 의존한 운영이 반복된다.

- **빈도**: 전체 요청 중 5% 미만에서만 피드백 수집
- **영향**: 서비스 품질 개선 기회 상실, 우수 사례 전파 불가
- **감정**: (관리자) 판단 근거 부재에 대한 불안

### 2.2 시장 기회 (Market Opportunity)

글로벌 Help Desk & Ticketing Software 시장은 2025년 기준 약 **USD 7.5B~15B** 규모이며, CAGR 9.5~12%로 성장 중이다. 특히 다음 트렌드가 ServiceDesk의 시장 기회를 뒷받침한다:

1. **AI 기반 자동화 수요 폭증**: 2026년 기준 91%의 서비스 리더가 AI 도입 압박을 받고 있으며, Agentic AI를 활용한 자율 해결(autonomous resolution)이 부상하고 있다.
2. **고객 승인 워크플로우 공백**: 기존 솔루션 대부분이 "완료 통보" 모델이며, 고객이 직접 결과를 승인하는 Approval Workflow를 핵심 기능으로 제공하는 플랫폼은 극소수이다.
3. **CSAT 통합 니즈**: 티켓 해결과 만족도 조사가 분리된 도구로 운영되는 경우가 많아, 통합 플랫폼에 대한 수요가 증가하고 있다.
4. **Mid-Market 서비스 격차**: ServiceNow는 Enterprise 전용(연간 라이선스 수천만 원), Zendesk/Freshdesk는 범용이지만 승인 워크플로우가 약하다. 50~500명 규모 조직을 위한 "적정 복잡도"의 솔루션이 부족하다.

**1. Zendesk**
- **강점**: 옴니채널 통합 최강, 풍부한 마켓플레이스(1,500+ 앱), 직관적 UI, 강력한 분석 도구

**3. Jira Service Management**
- **강점**: Jira/Confluence 통합, DevOps 파이프라인 연동, 개발팀과의 원활한 협업, 합리적 가격
- **약점**: 급경사 학습 곡선, 셀프서비스 포털 기능 제한, 대량 데이터 시 성능 저하, 비개발자 사용성 낮음
- **시장 점유율**: 마인드셰어 6.4% (2026.02 기준, 하락세)

**4. ServiceNow**
- **강점**: 엔터프라이즈 ITSM 최강, CMDB/ITOM 통합, Now Assist AI, 승인 워크플로우 내장, 강력한 자동화
- **약점**: 도입 비용 매우 높음(라이선스+구현 비용 3~5배), 도입 기간 3~6개월, 500명 미만 조직에는 과도한 복잡성
- **시장 점유율**: 마인드셰어 12.9% (2026.02 기준)

**5. HubSpot Service Hub**
- **강점**: CRM 네이티브 통합, 무료 도구 제공, 마케팅/영업/서비스 통합, 직관적 UI
- **약점**: 티켓 관리 기능 기본적, 고객 승인 워크플로우 없음, 복잡한 ITSM 시나리오 미지원, 엔터프라이즈 기능 제한
- **시장 점유율**: CRM 연계 서비스 세그먼트 강세

#### 경쟁 기회 (Competitive Opportunity)

ServiceDesk의 핵심 차별점: **고객 승인 워크플로우 (Approval Workflow)**

| Aspect | 기존 경쟁사 | ServiceDesk |
|--------|-------------|-------------|
| 완료 처리 | 지원팀이 일방적으로 "Resolved" 처리 | 고객이 직접 결과를 확인하고 "Approved" 또는 "Rejected" |

## 5. Product Vision & Strategy

### 5.1 Product Vision

> "모든 서비스 요청이 투명하게 처리되고, 고객이 직접 결과를 확인하며 신뢰가 쌓이는 세상"

ServiceDesk는 단순한 티켓 관리 도구가 아니라, **고객과 서비스 조직 간의 신뢰를 체계적으로 구축하는 플랫폼**이다. "요청 -> 배정 -> 처리 -> 고객 승인 -> 만족도 평가"라는 완결된 루프를 통해, 서비스의 모든 순간에서 투명성과 책임감을 제공한다.

#### SO 전략 (Strength + Opportunity)

- 한국 중견기업 디지털 전환 트렌드에 맞춰 "승인 워크플로우"를 핵심 메시지로 시장 선점
- AI 자동화를 점진적으로 도입하여 차별점 강화 (자동 배정 -> 자동 답변 -> 자율 해결)

#### WT 전략 (Weakness + Threat)

- 초기 고객 성공 사례를 빠르게 확보하여 레퍼런스 구축 (무료/할인 POC)
- 글로벌 경쟁사의 한국 진출에 대비하여 현지화 + 고객 밀착 지원으로 차별화

### 5.5 Porter's Five Forces Analysis

| Force | 강도 | 분석 |
|-------|------|------|
| **기존 경쟁자 간 경쟁** | **높음** | Zendesk, Freshdesk, Jira SM, ServiceNow, HubSpot 등 강력한 글로벌 플레이어 다수. 그러나 "고객 승인 워크플로우"를 핵심으로 내세우는 플레이어는 부재 |
| **신규 진입자 위협** | **중간** | SaaS 모델로 진입 장벽은 낮으나, 엔터프라이즈 고객 확보와 기능 완성도 확보에 시간 소요. AI 기반 신규 진입자(DevRev 등) 등장 |
| **대체재 위협** | **중간~높음** | 이메일, 스프레드시트, 메신저 등 비정형 대체재가 여전히 광범위하게 사용됨. "충분히 좋은(good enough)" 기존 방식의 관성 |
| **공급자 교섭력** | **낮음** | 클라우드 인프라(AWS/GCP/Azure) 경쟁으로 전환 비용 낮음. 오픈소스 기술 스택 활용 가능 |
| **구매자 교섭력** | **중간~높음** | 중견기업은 가격 민감도 높음. 경쟁 제품 간 전환 비용은 중간 수준(데이터 이전 필요)이나, 다수의 대안 존재 |

### 6.2 티켓 라이프사이클 (Ticket Lifecycle)

```
[고객 등록] -> [Open]
     |
     v
[자동/수동 배정] -> [Assigned]
     |
     v
[처리 중] -> [In Progress]
     |
     +---> [Pending] (고객 정보 대기)
     |        |
     |        v (고객 응답)
     |     [In Progress]
     |
     v
[처리 완료] -> [Resolved]
     |
     v
[고객 승인 요청] -> [Awaiting Approval]
     |
     +---> [Approved] -> [CSAT 평가] -> [Closed]
     |
     +---> [Rejected] (사유 입력) -> [Reopened] -> [In Progress]
```

## 8. GTM Strategy

### 8.1 Beachhead Segment (Geoffrey Moore -- Crossing the Chasm)

#### Beachhead Market 선정

| Criterion | Selection | Rationale |
|-----------|-----------|-----------|
| **Target Customer** | 한국 중견 제조업(100~500명)의 IT 서비스 운영팀 | 디지털 전환 가속 중이나 ServiceNow는 과도하고, Zendesk/Freshdesk는 현지화 부족. 내부 IT 서비스 데스크 필요성이 높으나 적절한 솔루션이 없는 세그먼트 |
| **Compelling Reason to Buy** | 고객(사내 직원) 불만 증가 + 경영진의 서비스 품질 가시성 요구 | 제조업 디지털 전환으로 IT 의존도 증가 -> IT 서비스 품질이 전사 생산성에 직결 -> 경영진이 "측정 가능한 서비스 품질"을 요구 |
| **Whole Product** | 티켓 관리 + 승인 워크플로우 + CSAT + SLA 대시보드 + 한국어 UI + 도입 컨설팅 | 도구만으로는 부족. 도입 지원(초기 설정, SLA 정책 설계, 워크플로우 구성)을 포함한 "완전한 제품" 제공 |
| **Competition** | 이메일/전화/엑셀 기반 수동 관리 (Direct Competitor 아닌 관행이 경쟁자) | 기존 ITSM 도구가 아닌, "비정형 수동 관리"에서 전환하는 것이 핵심. 처음 도입하는 고객에게 "첫 번째 서비스 데스크"로 자리매김 |

#### 4-Criteria Scoring

| Criterion | Score (1-5) | Evidence |
|-----------|:-----------:|----------|
| **고통의 강도 (Intensity of Pain)** | 5 | 제조업 IT 서비스 팀은 평균 일 30~50건의 비정형 요청을 이메일/전화로 처리. 추적 불가, 누락 빈번, 경영진 보고 불가 |
| **도달 가능성 (Reachability)** | 4 | 한국 제조업 IT 관리자 커뮤니티(KISA, ITAM 포럼), 전시회(IT EXPO Korea), SI 파트너 네트워크를 통해 접근 가능 |
| **구매력 (Budget)** | 4 | 중견 제조업 IT 예산 확보 가능. Agent당 $30~$50/월 수준이면 의사결정 부담 적음. 연간 계약 기준 $3,600~$6,000/10 agent |
